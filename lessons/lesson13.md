## 2. Понятие временного ряда

Временной ряд – это последовательность значений показателя, упорядоченных по времени.

Примеры временных рядов:

* выручка по месяцам;
* продажи по дням;
* количество клиентов по неделям;
* расходы компании по кварталам;
* спрос на товар по месяцам;
* посещаемость сайта по дням.

Ключевая особенность временного ряда – порядок наблюдений имеет значение.

Например:

```text
январь – 120000
февраль – 135000
март – 142000
апрель – 150000
```

Такие данные нельзя случайно перемешивать, потому что каждое значение связано с предыдущими периодами.

## 3. Основные компоненты временного ряда

Временной ряд может включать несколько компонентов:

* тренд – долгосрочное направление изменения показателя;
* сезонность – повторяющиеся колебания через одинаковые промежутки времени;
* случайные колебания – нерегулярные изменения, которые сложно заранее объяснить.

Пример:

```text
выручка постепенно растет – это тренд
выручка выше в декабре – это сезонность
резкое падение из-за разового события – это случайное колебание
```

Понимание структуры ряда помогает выбрать подход к прогнозированию.

## 4. Прогнозирование временного ряда

Прогнозирование временного ряда – это оценка будущих значений показателя на основе прошлых наблюдений.

Например:

```text
известна выручка за 36 месяцев
необходимо спрогнозировать выручку на следующие месяцы
```

В отличие от обычной регрессии, во временных рядах особенно важно учитывать порядок данных.

Нельзя обучать модель на будущих значениях и проверять ее на прошлых.

## 5. Подготовка данных

Для демонстрации используется искусственный набор данных с месячной выручкой.

```python
import pandas as pd
import numpy as np

np.random.seed(42)

dates = pd.date_range(
    start="2021-01-01",
    periods=48,
    freq="MS"
)

trend = np.arange(48) * 1500

seasonality = 8000 * np.sin(2 * np.pi * np.arange(48) / 12)

noise = np.random.normal(0, 3000, 48)

revenue = 100000 + trend + seasonality + noise

df = pd.DataFrame({
    "date": dates,
    "revenue": revenue
})

df.head()
```

В данном примере:

* `date` – дата наблюдения;
* `revenue` – выручка за месяц.

Данные содержат общий рост, сезонные колебания и случайный шум.

## 9. Лаговые признаки

Лаговый признак – это значение показателя в предыдущем периоде.

Например:

```text
revenue_lag_1 – выручка за предыдущий месяц
revenue_lag_2 – выручка два месяца назад
```

Лаги позволяют использовать прошлые значения ряда для прогноза будущего значения.

```python
df["revenue_lag_1"] = df["revenue"].shift(1)
df["revenue_lag_2"] = df["revenue"].shift(2)
df["revenue_lag_3"] = df["revenue"].shift(3)

df.head()
```

После создания лагов в первых строках появляются пропуски, потому что для первых месяцев нет предыдущих значений.

## 10. Признак месяца

Для учета сезонности можно добавить номер месяца.

```python
df["month"] = df["date"].dt.month

df.head()
```

Признак `month` помогает модели учитывать повторяющиеся изменения внутри года.

Например, если в декабре выручка обычно выше, модель может частично учесть это через номер месяца.

## 11. Формирование таблицы для модели

Для простой ML-модели используются лаговые признаки, скользящее среднее и номер месяца.

```python
df_model = df.dropna().copy()

features = [
    "revenue_lag_1",
    "revenue_lag_2",
    "revenue_lag_3",
    "rolling_mean_3",
    "month"
]

X = df_model[features]
y = df_model["revenue"]
```

В данном случае:

* `X` – признаки, рассчитанные на основе прошлых значений;
* `y` – выручка текущего месяца, которую необходимо спрогнозировать.

Важно: пропуски удаляются только после создания лагов и скользящих средних.

## 12. Разделение данных на train и test

Во временных рядах данные разделяются не случайно, а по времени.

Сначала идут обучающие данные, затем тестовые.

```python
train_size = int(len(df_model) * 0.8)

X_train = X.iloc[:train_size]
X_test = X.iloc[train_size:]

y_train = y.iloc[:train_size]
y_test = y.iloc[train_size:]
```

Такой подход имитирует реальную ситуацию:

```text
модель обучается на прошлом
модель проверяется на будущем
```

Случайное перемешивание данных для временного ряда может привести к утечке информации из будущего.


## 14. Оценка качества простой модели

Для оценки качества прогноза используются метрики регрессии.

Основные метрики:

* MAE – средняя абсолютная ошибка;
* RMSE – корень из среднеквадратичной ошибки;
* MAPE – средняя абсолютная процентная ошибка.

```python
from sklearn.metrics import mean_absolute_error, mean_squared_error

mae = mean_absolute_error(y_test, linear_pred)
rmse = np.sqrt(mean_squared_error(y_test, linear_pred))
mape = np.mean(np.abs((y_test - linear_pred) / y_test)) * 100

print("MAE:", round(mae, 2))
print("RMSE:", round(rmse, 2))
print("MAPE:", round(mape, 2), "%")
```

Интерпретация:

* MAE показывает среднюю ошибку прогноза в исходных единицах;
* RMSE сильнее учитывает крупные ошибки;
* MAPE показывает среднюю ошибку в процентах.

Например, если `MAPE = 8%`, это означает, что прогноз в среднем отклоняется от факта примерно на 8%.

## 16. Базовая модель для сравнения

Для проверки практической ценности модели ее необходимо сравнить с простым прогнозом.

Базовая модель может прогнозировать значение предыдущего месяца.

```python

```

Если ошибка линейной модели ниже ошибки базовой модели, значит модель дает дополнительную аналитическую ценность.

## 17. Понятие ARIMA

ARIMA – это классическая модель прогнозирования временных рядов.

Она использует три идеи:

* AR – зависимость текущего значения от прошлых значений;
* I – преобразование ряда для устранения тренда;
* MA – учет прошлых ошибок прогноза.

Обозначение модели:

```text
ARIMA(p, d, q)
```

Где:

* `p` – сколько прошлых значений ряда используется;
* `d` – сколько раз берется разность ряда для устранения тренда;
* `q` – сколько прошлых ошибок прогноза учитывается.

ARIMA подходит для рядов, где важны прошлые значения и тренд, но она не всегда хорошо учитывает выраженную сезонность.

## 18. Понятие SARIMA

SARIMA – это расширение ARIMA, которое дополнительно учитывает сезонность.

Обозначение модели:

```text
SARIMA(p, d, q)(P, D, Q, s)
```

Первая часть отвечает за обычную динамику ряда:

* `p` – прошлые значения;
* `d` – обычное дифференцирование;
* `q` – прошлые ошибки.

Вторая часть отвечает за сезонную динамику:

* `P` – сезонные прошлые значения;
* `D` – сезонное дифференцирование;
* `Q` – сезонные ошибки;
* `s` – длина сезонного цикла.

Например:

```text
s = 12
```

Для месячных данных это означает годовую сезонность, так как в году 12 месяцев.

SARIMA особенно полезна, если показатель имеет повторяющиеся сезонные колебания.

```python
df_ts.head()
```

## 20. Разделение ряда на train и test для SARIMA

Для SARIMA также используем разделение по времени.

```python
test_size = 6

train_ts = df_ts.iloc[:-test_size]
test_ts = df_ts.iloc[-test_size:]
```

## 21. Обучение SARIMA

Для построения SARIMA используется класс `SARIMAX` из библиотеки `statsmodels`.

```python
from statsmodels.tsa.statespace.sarimax import SARIMAX

sarima_model = SARIMAX(
    train_ts["revenue"],
    order=(1, 1, 1),
    seasonal_order=(1, 1, 1, 12),
    enforce_stationarity=False,
    enforce_invertibility=False
)

sarima_result = sarima_model.fit(disp=False)
```

Здесь используется модель:

```text
SARIMA(1, 1, 1)(1, 1, 1, 12)
```

Для первого знакомства параметры можно воспринимать как базовую настройку для месячного ряда с годовой сезонностью.

На практике параметры модели подбираются отдельно, но на начальном уровне достаточно понять общую логику.

## 22. Прогноз SARIMA на тестовый период

```python
    "date": test_ts.index,
    "actual": test_ts["revenue"],
    "predicted": sarima_pred
})

sarima_result_df.head()
```

Где:

* `actual` – фактическая выручка;
* `predicted` – прогноз SARIMA.

## 23. Оценка качества SARIMA

Оценим качество прогноза теми же метриками.

```python
sarima_mae = mean_absolute_error(
    sarima_result_df["actual"],
    sarima_result_df["predicted"]
)

sarima_rmse = np.sqrt(
    mean_squared_error(
        sarima_result_df["actual"],
        sarima_result_df["predicted"]
    )
)

sarima_mape = np.mean(
    np.abs(
        (sarima_result_df["actual"] - sarima_result_df["predicted"])
        / sarima_result_df["actual"]
    )
) * 100

print("SARIMA MAE:", round(sarima_mae, 2))
print("SARIMA RMSE:", round(sarima_rmse, 2))
print("SARIMA MAPE:", round(sarima_mape, 2), "%")
```

Интерпретация остается такой же:

* чем ниже MAE, тем меньше средняя ошибка;
* чем ниже RMSE, тем меньше крупные ошибки;
* чем ниже MAPE, тем ниже средняя процентная ошибка.

## 26. Прогноз SARIMA на будущий период

После проверки модели можно обучить SARIMA на всем доступном ряде и построить прогноз на будущие месяцы.

```python
    "date": future_dates,

future_df
```

Этот прогноз строится на основе всей доступной истории.

## 28. Когда использовать SARIMA

SARIMA целесообразно использовать, если:

* данные представлены в виде регулярного временного ряда;
* есть достаточно длинная история наблюдений;
* в ряде присутствует сезонность;
* необходимо получить прогноз на несколько будущих периодов;
* важна классическая и интерпретируемая модель временного ряда.

SARIMA менее удобна, если:

* данные сильно нерегулярные;
* много пропусков;
* временной ряд очень короткий;
* на показатель сильно влияют внешние факторы, которых нет в модели;
* структура ряда часто меняется.

## 29. Типичные ошибки при прогнозировании временных рядов

| Ошибка                                      | Почему это проблема                                       |

| Случайное разделение данных на train и test | Возникает утечка информации из будущего                   |
| Использование будущих значений в признаках  | Модель получает недоступную в реальности информацию       |
| Оценка качества только на обучающей выборке | Качество прогноза может быть искусственно завышено        |
| Отсутствие сравнения с базовой моделью      | Непонятно, дает ли модель практическую пользу             |
| Игнорирование сезонности                    | Модель может плохо прогнозировать повторяющиеся колебания |
| Неверное значение сезонного периода `s`     | SARIMA будет учитывать неправильный сезонный цикл         |
| Прогноз на слишком длинный горизонт         | Ошибка прогноза может быстро накапливаться                |
| Использование MAPE при нулевых значениях    | Деление на ноль делает метрику некорректной               |
