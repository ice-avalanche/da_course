# Урок 8. Очистка и подготовка данных

## 1. Цель занятия

Освоить методы подготовки данных и научиться:

* находить ошибки в данных;
* обрабатывать пропуски;
* исправлять типы данных;
* удалять дубликаты;
* работать с выбросами;
* подготавливать данные для анализа и моделей.

## 2. Зачем нужна очистка данных

В реальных данных почти всегда есть ошибки.

Пример:

```python
import pandas as pd

df = pd.DataFrame({
    "revenue": [100, 200, None, 150],
    "cost": ["70", "120", "90", None]
})

df["profit"] = df["revenue"] - df["cost"]
```

Ошибка:

```
TypeError
```

После очистки:

```python
df["cost"] = df["cost"].astype(float)
df = df.dropna()

df["profit"] = df["revenue"] - df["cost"]
print(df)
```

Ключевая идея: получают сырые данные → проводят очистку → проводят анализ → используют в модели.

## 3. Типовые проблемы данных

### 3.1 Пропуски

Пропуски возникают, когда данные отсутствуют или не были записаны. Это одна из самых частых проблем в реальных датасетах.

```python
df.isna()
df.isna().sum()
```

Проблема: наличие пропусков мешает корректным вычислениям.

### 3.2 Неверные типы

Тип данных определяет, какие операции можно выполнять с колонкой. Ошибки типов часто возникают при загрузке данных из файлов.

```python
df["revenue"] + 10
```

Проблема: строковые значения нельзя использовать в числовых вычислениях, это приводит к ошибкам при выполнении.

### 3.3 Дубликаты

Дубликаты – это повторяющиеся строки, которые могут появляться при объединении или загрузке данных.

```python
df.duplicated()
```

Проблема: наличие дубликатов приводит к завышению или занижению агрегированных показателей, таких как сумма и среднее.

### 3.4 Выбросы

Выбросы – это значения, сильно отличающиеся от остальных наблюдений.

```python
df = pd.DataFrame({"revenue": [100, 110, 120, 5000]})
```

Проблема: выбросы существенно искажают статистические показатели, делая их нерепрезентативными.

### 3.5 Несогласованные категории

Категориальные признаки могут содержать разные варианты записи одного и того же значения.

```python
df = pd.DataFrame({"region": ["Москва", "москва", "Moscow"]})
```

Проблема: различные формы записи одной категории интерпретируются как разные значения, что искажает результаты анализа.

### 3.6 Логические ошибки

Логические ошибки – это значения, которые противоречат здравому смыслу или бизнес-логике.

```python
df = pd.DataFrame({"revenue": [100, -50, 200]})
```

Проблема: наличие логически некорректных значений приводит к ошибочным выводам и недостоверным результатам анализа.

## 4. Работа с пропусками

Пропуски необходимо либо удалить, либо заменить на корректные значения. Выбор метода зависит от задачи и объема данных.

### 4.1 Удаление

Подходит, если пропусков немного и их удаление не влияет на структуру данных.

```python
df.dropna()
df.dropna(subset=["revenue"])
```

### 4.2 Заполнение

Используется, когда важно сохранить все наблюдения.

Средним:

```python
df["revenue"] = df["revenue"].fillna(df["revenue"].mean())
```

Медианой:

```python
df["cost"] = df["cost"].fillna(df["cost"].median())
```

Константой:

```python
df["cost"] = df["cost"].fillna(0)
```

Категории:

```python
df["region"] = df["region"].fillna("unknown")
```

### 4.3 Для временных данных

В временных рядах часто используют заполнение предыдущим значением.

```python
df["value"] = df["value"].fillna(method="ffill")
```

## 5. Преобразование типов данных

Типы данных должны соответствовать содержанию колонок. Это необходимо для корректных вычислений и анализа.

### 5.1 Проверка

```python
df.dtypes
```

Позволяет быстро определить, какие колонки требуют преобразования.

### 5.2 Числа

```python
df["revenue"] = df["revenue"].astype(float)
```

Используется, если данные уже корректны, но имеют неверный тип.

Безопасно:

```python
df["revenue"] = pd.to_numeric(df["revenue"], errors="coerce")
```

Позволяет автоматически заменить некорректные значения на NaN.

### 5.3 Даты

```python
df["date"] = pd.to_datetime(df["date"])
```

Преобразование строк в формат даты необходимо для работы с временными признаками.

### 5.4 Работа с датами

```python
df["year"] = df["date"].dt.year
df["month"] = df["date"].dt.month
```

Позволяет извлекать компоненты даты для анализа.

## 6. Дубликаты

Дубликаты искажают результаты анализа и должны быть обработаны.

### 6.1 Поиск

```python
df.duplicated()
```

Возвращает строки, которые повторяются.

### 6.2 Удаление

```python
df.drop_duplicates()
```

Удаляет повторяющиеся записи.

По колонке:

```python
df.drop_duplicates(subset=["id"])
```

Используется, если уникальность определяется конкретным признаком.

### 6.3 Варианты

```python
keep="first"
keep="last"
keep=False
```

Позволяют управлять тем, какие дубликаты сохранять.

## 7. Выбросы

Выбросы требуют анализа: их можно удалить, ограничить или оставить.

### 7.1 Поиск

```python
df.describe()
```

Позволяет выявить аномальные значения через статистику.

### 7.2 IQR

Метод межквартильного размаха используется для выявления выбросов на основе распределения данных.

```python
Q1 = df["revenue"].quantile(0.25)
Q3 = df["revenue"].quantile(0.75)

IQR = Q3 - Q1

lower = Q1 - 1.5 * IQR
upper = Q3 + 1.5 * IQR

df = df[(df["revenue"] >= lower) & (df["revenue"] <= upper)]
```

> Q1 – значение, ниже которого находится 25% данных.
> 
> Q3 – значение, ниже которого находится 75% данных.
> 
> IQR (межквартильный размах) показывает диапазон, в котором находится центральная часть данных.
> 
> Границы lower и upper определяют допустимый диапазон значений: все, что выходит за эти пределы, считается выбросом.
> 
> Коэффициент 1.5 – эмпирическое правило, используемое для отделения аномальных значений от основной массы данных.

### 7.3 Ограничение

```python
df["revenue"] = df["revenue"].clip(upper=200)
```

Позволяет ограничить экстремальные значения без удаления строк.

### 7.4 Логическое правило

```python
df = df[df["revenue"] >= 0]
```

Используется, если есть бизнес-ограничения.

## 8. Feature engineering

Создание новых признаков позволяет улучшить качество анализа и моделей.

### 8.1 Метрики

```python
df["profit"] = df["revenue"] - df["cost"]
df["margin"] = df["profit"] / df["revenue"]
```

Позволяет получить производные показатели.

### 8.2 Биннинг

```python
df["group"] = pd.cut(
    df["revenue"],
    bins=[0, 120, 180, 300],
    labels=["low", "medium", "high"]
)
```

Используется для перевода чисел в категории.

### 8.3 Даты

```python
df["month"] = df["date"].dt.month
```

Позволяет анализировать сезонность.

### 8.4 Логические признаки

```python
df["high_revenue"] = df["revenue"] > 150
```

Удобны для сегментации.

### 8.5 Группировки

```python
df["avg_revenue"] = df.groupby("region")["revenue"].transform("mean")
```

Позволяет добавить агрегированные характеристики.
